## 5. 페이지별 개발 명세

### 메인 페이지 (`/`)
- GNB: 엠블럼 로고 + 8개 메뉴 드롭다운 + 로그인/회원가입
- 히어로 배너: 풀스크린 슬라이드 (CMS 관리)
- 퀵 메뉴: 인하상회/사랑카드/플레이스/동문검색/기부 5개 아이콘
- 소식: 공지사항/주요행사/동문보도 탭 전환 최신 5개
- 주요 사업: 4개 카드 (상회/카드/플레이스/콘서트)
- 기부 대시보드: 3개 기금 프로그레스 바 + 최근 기부자 롤링
- 포토 갤러리: 최신 4~8장 그리드
- 푸터: 주소/연락처/표어/저작권

### 서브 페이지 공통 레이아웃
- 브레드크럼 (홈 > 카테고리 > 페이지)
- 좌측 사이드 메뉴 (현재 카테고리 하위 메뉴)
- 우측 콘텐츠 영역

### 게시판 공통 스펙
- 목록: 번호/태그/제목/등록일/조회수 + 페이지네이션 + 검색
- 상세: 제목/작성일/본문/첨부파일/이전글/다음글
- 관리자: 작성/수정/삭제/공지고정/팝업설정

### 인하플레이스 (`/business/place`)
- Kakao Map 전체 표시
- 카테고리 필터: 요식업/의료/사무소/교육/서비스/기타
- 거리순/평점순 정렬
- 업소 카드: 이름/카테고리/주소/동문우대혜택
- 업소 등록 (인증 동문 — CMS에서 승인)

### 기부/결제 (`/donate`)
- 기금 유형 선택: 회비발전기금/장학기금/건립기금
- 금액 프리셋: 10만/50만/100만/직접입력
- Toss Payments 위젯 연동
- 약정(정기결제) 시스템
- 계좌이체 안내: 우리은행 256-454416-13-001

## 7. 개발 순서

### Phase 1 (Day 1-2): 기본 골격
프로젝트 초기화, Tailwind 설정, 레이아웃, 메인 페이지, 반응형

### Phase 2 (Day 3-4): 정적 페이지
총동창회 소개 5p, 사업 소개 5p, 장학회 4p, 모교소식 2p, 오시는길

### Phase 3 (Day 5-7): 게시판/CRUD
Supabase 연동, 게시판 컴포넌트, 6개 게시판, 동창회보 PDF, 초기 데이터 시딩

### Phase 4 (Day 8-12): 핵심 기능
회원가입/로그인, 동문인증, 동문검색, 인하플레이스, 기부/결제, 기부 대시보드

### Phase 5 (Day 13-16): 관리자/부가
CMS, 배너관리, 메일/문자, 뉴스레터, 멘토링

### Phase 6 (Day 17-20): 테스트/배포
Vercel 배포, QA, Lighthouse, SEO, 도메인 연결
